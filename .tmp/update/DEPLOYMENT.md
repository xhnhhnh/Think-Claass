# 项目部署指南

本文档旨在为您提供详尽的步骤，帮助您在全新的服务器环境中从零开始部署本项目的生产环境。

## 1. 环境准备

在开始部署之前，请确保您的服务器已经安装并配置了以下必要的运行环境：

- **Node.js** (推荐使用 v22.x 版本)
- **npm** (通常与 Node.js 一同安装)
- **PM2** (用于管理 Node.js 进程，确保服务在后台稳定运行)

**安装 Node.js 和 npm (以 Ubuntu/Debian 为例):**
```bash
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt-get install -y nodejs
```

**全局安装 PM2:**
```bash
npm install -g pm2
```

## 2. 下载与解压项目文件

将打包好的 `project.zip` 上传至您的服务器，然后解压到您希望部署的目录中（例如 `/www/wwwroot/project`）。

```bash
# 创建项目目录并进入
mkdir -p /www/wwwroot/project
cd /www/wwwroot/project

# 解压项目文件 (请确保服务器已安装 unzip 命令)
unzip project.zip
```

## 3. 安装项目依赖

进入项目根目录后，运行以下命令安装生产环境所需的依赖包：

```bash
npm install --production
```
> **注意：** 使用 `--production` 参数可以跳过开发依赖（devDependencies）的安装，从而加快安装速度并减少磁盘占用。

## 4. 启动服务

依赖安装完成后，使用 PM2 启动项目服务。项目默认提供了 PM2 配置文件：

```bash
pm2 start ecosystem.config.cjs
```

启动后，服务默认会在本地的 `3000` 端口运行（即 `http://127.0.0.1:3000`）。您可以运行以下命令来查看服务状态并设置开机自启：

```bash
pm2 status       # 查看服务运行状态
pm2 save         # 保存当前运行列表
pm2 startup      # 设置 PM2 开机自启
```

## 5. 配置反向代理

为了让外部用户可以通过域名和标准端口（HTTP 80 / HTTPS 443）访问您的服务，建议使用反向代理。以下是主流服务器管理面板的配置方法：

### 宝塔面板 (BT Panel) 配置方法
1. 登录宝塔面板，进入 **“网站”** -> **“添加站点”**。
2. 填写您的域名，根目录可随意指定或留空，PHP版本选择 **“纯静态”**，点击提交。
3. 在网站列表中找到刚刚添加的站点，点击 **“设置”**。
4. 在左侧菜单中选择 **“反向代理”** -> **“添加反向代理”**。
5. 填写以下信息：
   - **代理名称**：随意填写（如 `node_proxy`）
   - **目标URL**：填写 `http://127.0.0.1:3000`
   - **发送域名**：填写 `$host`
6. 点击提交并保存。现在您可以通过域名访问网站了。如果需要 HTTPS，可以在“SSL”选项卡中申请或部署证书。

### 1Panel 面板配置方法
1. 登录 1Panel 面板。
2. 进入 **网站** -> **创建网站** -> **反向代理**。
3. 填写以下信息：
   - **主域名**：填写您的主域名
   - **代理地址**：设置代理地址为 `127.0.0.1:3000`
4. 点击确认创建。
5. （可选）设置 SSL 以启用 HTTPS 访问。

## 6. 默认管理员凭证

系统启动并成功访问后，您可以使用以下默认凭证登录后台管理系统：

- **用户名:** `admin`
- **密码:** `123456`

> **⚠️ 强烈建议:** 首次登录系统后，请务必立即前往后台管理界面修改默认的管理员密码，以确保系统和数据安全！
