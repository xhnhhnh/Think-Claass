# Feature: 所有的功能都可以在教室当中开关

## 🎯 需求背景 (Background)
目前系统中，教师若要管理班级特定功能（如商品兑换、宠物精灵、兑换记录等），必须离开当前的教学场景（例如大屏展示、主控台），跳转到专门的**“功能开关”**页面 (`/teacher/features`) 进行设置。
在实际课堂教学中，教师经常需要根据课堂进度临时打开或关闭某些互动功能或教学工具（如抽奖、点名、倒计时等）。频繁切换页面会打断教学节奏，导致课堂体验不连贯。

## 💡 核心目标 (Goal)
实现**“所有的功能（除超级管理员专属设置外）都可以在教室（教学场景主界面）当中直接开关”**。
让教师能够在授课的主界面（如主控台 Dashboard、大屏展示 Bigscreen 或教学工具侧边栏）上，直接快速地切换各种功能的启用/停用状态。

## 🛠 详细需求说明 (Detailed Requirements)

### 1. 现有系统开关整合
需要将当前位于 `/teacher/features` 中的所有开关，无缝集成到“教室”场景中。包含但不限于：
- [ ] 商品兑换 (`enableShop`)
- [ ] 宠物精灵系统 (`enablePets`)
- [ ] 兑换记录查询 (`enableRecords`)

### 2. 教学工具开关整合
课堂互动常用的教学辅助工具也应支持一键开关，便于统一管理课堂秩序：
- [ ] 随机点名
- [ ] 倒计时 / 秒表计时
- [ ] 随机分组
- [ ] 抽奖 / 九宫格

### 3. 所有互动功能模块开关
所有课堂互动模块应提供快捷启用/停用的入口：
- [ ] 团队任务 (Team Quests)
- [ ] 跨班大乱斗 (Brawl)
- [ ] 世界 BOSS (World Boss)
- [ ] 盲盒与拍卖行 (Blind Box & Auction)
- [ ] 领土扩张 (Territory)

## 💻 建议的实现方案 (Proposed Implementation)
1. **统一的快捷面板 (Quick Settings Panel)**：
   在“班级与学生管理 (Dashboard)”和“大屏展示 (Bigscreen)”页面中，增加一个浮动的“课堂功能管理面板”或侧边栏抽屉 (Drawer)。
2. **状态实时同步**：
   复用现有的 `/api/classes/:classId/settings` 接口，确保快捷面板中的操作能够实时保存并下发生效，无需刷新页面。
3. **UI/UX 设计**：
   设计简洁的 Switch 开关组件，使用与当前平台一致的 Tailwind 样式，确保交互顺滑且视觉不突兀。

## ✅ 验收标准 (Acceptance Criteria)
- [ ] 教师能够在“大屏展示”或“主控台”页面中一键呼出功能开关面板。
- [ ] 面板中包含了上述所有列出的功能模块开关。
- [ ] 开启或关闭功能后，学生端及相应模块的功能入口能够实时显示或隐藏。
- [ ] 操作体验流畅，且不会干扰正在进行的课堂互动活动。
